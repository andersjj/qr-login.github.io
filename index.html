No Passwords

Simply put, passwords are a thing of the past. We have the technology to authenticate other ways, and we should move towards that. It's easy to get content with passwords, but we forget how risky they can be. Now is the time to evolve how we authenticate to websites and other online systems.

Secure

Passwords are bad in that they are very static. Even encrypted/hashed passwords only change when the owner changes them. This gives hackers much more time to breach them. QR-Login's authentications are only valid for 30 seconds, making it much harder for hackers to breach it.

Transparent

QR-Login is built on industry accepted standards and practices for digital signing, encryption where needed and secure authorization. This means that we are very open about how QR-Login works with digital keys without comprimising the security of transactions.

Secure

QR-Login uses industry accepted private/public key functionality at it's core to provide more secure authorization than just passwords. It also incorporates a rotating TOTP code and a random nonce to ensure that replay attacks won't happen. All this security doesn't hurt the user experience either. All of these security elements can be regenerated at will with a single tap, further enhancing security.

Easy to use

With all this security, you'd think that QR-Login would be cumbersome to use, and nothing could be farther from the truth. You can setup a QR-Login account on any website that supports it with a few taps, and login with one single tap. All the work is done behind the scenes by the app. In a situation like at a hotel, trying to login to a streaming service, you can use QR-Login to authenticate with a single scan. Much better than trying to type in your username and password on an on-screen keyboard and remote.

Choice is good

If you've read this far, you're probably saying "Isn't this just like Passkeys ?".   QR-Login is based on the same public/private keys, and QR codes, but it's much more. It has TOTP functionality built in for extra security and has more options. You're able to turn off the cross-device sync to keep those keys on your device, and only on your device. It also allows you to update the keys with a single tap, allowing you to change the certs on your schedule to further enhance security.

Cross Platform/Device

Other solutions (such as Passkeys) are similar to how QR-Login works, especially the public-private key implementation, so QR-Login is as secure as Passkeys. However, Passkeys are tightly integrated into the platform you're running on. QR-Login is designed from the ground up to work flexibly across all platforms. That also extends to IOT devices, Raspberry Pi, door locks, security gates, etc. Anything that can display a QR code and make an HTTP call can work with QR-Login.
